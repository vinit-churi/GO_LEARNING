# Phase 2 Plan: Intermediate

Status: `pending`

## Goal

Move from syntax fluency to idiomatic program structure, standard library usage, and concurrency fundamentals.

## Scope

This phase should cover:

1. pointers and semantics
2. composition and embedding
3. interfaces in practice
4. custom errors and wrapping
5. standard library essentials
6. file I/O
7. JSON and encoding
8. time and utility packages
9. modules and package design
10. table-driven testing
11. benchmarks and fuzzing
12. goroutines
13. channels
14. select, timeouts, and cancellation
15. context
16. mutexes, atomics, and race detection
17. HTTP clients
18. HTTP servers

## Recommended Build Order

Implement in batches:

1. program structure and package design
2. standard library and data handling
3. concurrency foundations
4. networking and application boundaries

## Quality Bar

Intermediate lessons should begin emphasizing:

- idiomatic naming
- package boundaries
- test readability
- practical tradeoffs
- debugging and observability basics

## Completion Criteria

Phase 2 is complete when all scope modules have planned lesson folders and implemented content that follows the content spec.
